# Итерация 1 — каркас сайта (media-site)

> Детальная спецификация для Claude Code. Разворачивает Итерацию 1 из `08_Итерации_разработки.md` с учётом дизайн-решений, зафиксированных в сессии 03.07.2026 (см. `CLAUDE.md` и `mockups/site-mockup.html`).
>
> **Визуальный источник истины — `mockups/site-mockup.html`.** Всё, что ниже, — как перенести этот макет в настоящий Astro-проект. Где текст расходится с макетом, приоритет у макета.

---

## Цель итерации

Инициализировать Astro-проект `media-site`, перенести дизайн-систему из макета, собрать базовые страницы (главная + материал) на тестовом контенте. На выходе — локально запускается сайт, визуально совпадающий с макетом.

## Критерий приёмки

- [ ] `npm run dev` и `npm run build` проходят без ошибок.
- [ ] Главная и страница материала визуально совпадают с `mockups/site-mockup.html` (палитра «Петроль», текст-first, serif-заголовки).
- [ ] Content Collections типизированы; тестовый `.md` проходит валидацию.
- [ ] Светлая/тёмная тема работают, переключаются по `prefers-color-scheme`.
- [ ] Нет console errors; мобильная вёрстка (одна колонка, бургер) корректна.
- [ ] Lighthouse Performance ≥ 90 (Astro статика, шрифты self-hosted).

---

## Предпосылки

- Репозиторий `media-site` создан (Этап 0). Если нет — создать пустой репо и склонировать.
- Node LTS. Пакетный менеджер — npm.
- Документ `mockups/site-mockup.html` из репо `media` скопирован в `media-site/design/` как референс.

## Шаг 1. Инициализация

```bash
npm create astro@latest . -- --template minimal --install --typescript strict
npx astro add tailwind --yes
npm install @fontsource-variable/source-serif-4 @fontsource-variable/inter
```

> ⚠️ **Шрифты — не системные, как в макете, а реальные из ТЗ 02**: Source Serif 4 (serif, тело+заголовки) и Inter (sans, служебное). Подключать через Fontsource (self-hosting, без Google Fonts CDN). Проверить кириллицу у обоих (поддерживается).

`astro.config.mjs`: `site: 'https://<домен>'`, интеграции tailwind + sitemap (sitemap подключим в Итерации 3), Markdown shiki theme.

## Шаг 2. Дизайн-токены

Перенести CSS-переменные из макета в `src/styles/global.css` (`:root`, `@media (prefers-color-scheme: dark)`, `:root[data-theme=...]`). Значения — ровно из макета:

### Светлая тема
```
--paper:#F7F5F0  --paper-alt:#F0EDE5  --paper-sunk:#E9E5DB  --surface:#FCFBF8
--ink:#26262A    --ink-soft:#585862   --ink-muted:#8F8D8C
--line:#E3DED3   --line-strong:#D2CCBE
--accent:#14636B (Петроль)  --accent-hover:#0F4C52  --accent-tint: ~10% accent на paper
```
### Тёмная тема
```
--paper:#17150F  --paper-alt:#1E1B14  --paper-sunk:#131009  --surface:#211D16
--ink:#ECE7DD    --ink-soft:#B6AFA1   --ink-muted:#8B8477
--line:#2F291F   --line-strong:#3D3627
--accent: осветлённый Петроль (в макете color-mix 58% accent + #EFE9DD)
```

### Типографика
| Роль | Шрифт | Размер |
|---|---|---|
| H1 hero (главная) | Source Serif | `clamp(2.3rem, 4.2vw, 3.4rem)` / 1.1 |
| H1 статьи | Source Serif | `clamp(2.1rem, 4.6vw, 3.05rem)` / 1.13 |
| Лид | Source Serif (в статье — italic) | 20px / 1.6 |
| Тело статьи | Source Serif | 19px / 1.72 |
| Заголовок в ленте | Source Serif bold | 24px / 1.24 |
| H2/H3 в статье | Inter | 1.5rem / 1.2rem |
| Навигация, мета, подписи | Inter | 11–14px |
| Дата чтения, теги-моно | mono | 12.5px |

### Сетка
Контейнер `max-width: 1160px`; колонка статьи `720px`; радиус 6px; тени минимальные. Всё — как в макете.

> Токены удобно завести и в `tailwind.config.mjs` (colors/​fontFamily/​maxWidth), чтобы верстать утилитами, но цвета держать источником в CSS-переменных (для тем).

## Шаг 3. Content Collections

`src/content/config.ts` — схема из `08_Итерации_разработки.md` (Итерация 1). Ключевое:
- Коллекции: `news`, `digest`, `reviews`, `columns`, `reports`, `authors`.
- Фронт-маттер: `title, description(≤160), pubDate, type, category, geo[], tags[], author?, featured, cover?, source?`.
- `category` остаётся (нужен для URL `/category/[slug]` и фильтров), **но цветовой кодировки категорий нет** — это только навигация. Гео (`geo`) выводится маленьким нейтральным тегом `РФ/МИР/АЗИЯ`.

## Шаг 4. Компоненты (текст-first)

Перенести из макета один в один. Картинок в карточках нет.

| Компонент | Назначение |
|---|---|
| `BaseLayout.astro` | `<html data-theme>`, `<head>` (мета из Шага 2 Итерации 3), Header + `<slot>` + Footer |
| `Header.astro` | Sticky-шапка: лого «Охват.», дата-строка, навигация, иконка поиска, CTA «Подписаться» (акцент), бургер на мобайле |
| `Footer.astro` | Многоколоночный: бренд, Разделы, Гео, Медиа, соцсети |
| `CategoryEyebrow.astro` | Надзаголовок капсом (акцент) + гео-тег. Без цветной заливки |
| `ArticleListItem.astro` | Элемент ленты «Свежее»: eyebrow → serif-заголовок → dek → мета. **Только текст** |
| `HeroFeature.astro` | Текстовый hero: eyebrow → крупный H1 → лид → мета. Проп `image?` — **опциональная** картинка только здесь, по умолчанию off |
| `SideList.astro` | Блок «Главное сейчас» (список текстовых заголовков) |
| `RailDigest.astro` | Сайд-бар: «Дайджест недели» (заголовок + булеты) |
| `RailPopular.astro` | Сайд-бар: «Популярное» — нумерованный 1–5 (номера осмысленны — рейтинг) |
| `OpinionCard.astro` | Карточка колонки: цитата + автор (аватар-инициалы) |
| `AuthorBio.astro` | Мини-карточка автора под статьёй |
| `RelatedList.astro` | «Читайте также» — 3 текстовые карточки |
| `NewsletterCTA.astro` | Форма подписки (без бэкенда — заглушка) |

**Спец-блоки тела статьи** (как MDX-компоненты): `Callout`, `StatBlock`, сравнительная таблица (`.cmp-table` в `overflow-x:auto`), `SourceBox`, `TagList`. Стили — из макета.

## Шаг 5. Страницы

| Файл | URL | Содержимое |
|---|---|---|
| `pages/index.astro` | `/` | Hero (featured) + «Главное сейчас» → две колонки (лента «Свежее» + сайд-бар Дайджест/Популярное) → «Мнения» → подписка |
| `pages/article/[...slug].astro` | `/[slug]` | Хлебные крошки → категория → H1 → лид → мета → тело (720px) со спец-блоками → автор → «Читайте также» → подписка |
| `pages/news/index.astro` и др. | `/news`, `/digest`, `/reviews`, `/reports`, `/columns` | Ленты по типу |
| `pages/author/[slug].astro` | `/author/[slug]` | Профиль автора + его материалы |
| `pages/about.astro` | `/about` | Статика |
| `pages/404.astro` | — | 404 |

Разметка страниц — точный перенос секций из макета (структуры главной и статьи там уже собраны).

## Шаг 6. Тестовый контент

3 файла вручную (для непустого сайта):
- `content/columns/zachem-my-zapuskaem-media-pro-adtech.md` — колонка основателя.
- `content/news/yandex-native-rsy-2026.md` — новость (текст из макета годится как рыба).
- `content/authors/y-sokolov.md` — профиль автора (name, role, bio, social).

---

## Отличия от исходного ТЗ 02 (важно, не откатывать)

- Акцент **`#14636B` (Петроль)**, а не red-orange `#C8472B`.
- **Нет цветовой кодировки категорий** — категории типографские, гео — нейтральный тег.
- **Карточки без превью-картинок** — контент текстовый; картинка опциональна только на featured.
- Serif-forward: serif и в заголовках карточек, не только в теле.
