## 1. Архитектура системы (общая схема)

```
┌────────────────────────────────────────────────────────────────┐
│  РЕПОЗИТОРИЙ 1: media-agents  (Python, отдельный репо)          │
│                                                                │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐  │
│  │ Сборщик  │ →  | Фильтр + │ →  | Автор    │ →  | Публика- │  │
│  | (parse)  │    | дедуп    │    | (LLM)    │    | тор      │  │
│  └──────────┘    └──────────┘    └──────────┘    └─────┬────┘  │
│       ↑                                                 │       │
│       │ Cron / GitHub Actions запускает по расписанию   │       │
└───────┼─────────────────────────────────────────────────┼──────┘
        │                                                 │
        │ RSS, парсинг                                    | git push
        │ пресс-релизов,                                  | / PR
        │ блогов, фин.                                    ↓
        │ отчётов               ┌──────────────────────────────────┐
        │                       │  РЕПОЗИТОРИЙ 2: media-site       │
        │                       │  (Astro + Markdown)              │
        │                       │                                  │
        └──────────────────────→│  src/content/news/*.md           │
                                │  src/content/columns/*.md        │
                                │  src/content/reports/*.md        │
                                │                                  │
                                │  → Vercel/Cloudflare Pages       │
                                │    (автопубликация при пуше)     │
                                └──────────────────────────────────┘
```

**Два отдельных репозитория — критично:**
- `media-agents` — код агентов, ломается → сайт продолжает работать.
- `media-site` — статический сайт на Astro, контент в Markdown.
- Агенты коммитят в `media-site` через git (PR для аналитики, прямой пуш в main для коротких новостей).

## 2. Сайт (`media-site`)

### Фреймворк
- **Astro** (рекомендованная версия — последняя стабильная).
- Content Collections для типизации Markdown-фронт-маттера.
- Island architecture для интерактивных элементов (форма подписки, поиск).

### Хранение контента
- **Markdown-файлы** в `src/content/`, разделённые по коллекциям:
  - `src/content/news/` — короткие агентские новости
  - `src/content/digest/` — еженедельные/ежедневные дайджесты
  - `src/content/reviews/` — обзоры инструментов и продуктов
  - `src/content/columns/` — экспертные колонки (человеческие авторы)
  - `src/content/reports/` — глубокие аналитические отчёты (как detailed.com Reports)
  - `src/content/authors/` — данные об авторах (JSON/MD)
- Фронт-маттер стандартизирован — схема в `src/content/config.ts`.

### Стилизация
- **Tailwind CSS** + `@tailwindcss/typography` для типографики статей.
- Дизайн-токены (цвета, шрифты, отступы) в `tailwind.config.mjs`.

### Шрифты (важно — типографика как в Digiday)
- Заголовки: системный grotesk (Inter / Manrope / Geist Sans) — на русском читается хорошо.
- Текст: serif для статей (Source Serif Pro / Lora / Literata) — даёт «журнальность» как у Digiday.
- Подобрать точные шрифты в `02_Структура_и_дизайн_сайта.md`.

### Деплой
- **Vercel** (первый выбор) или **Cloudflare Pages** (если нужен бесплатный bandwidth).
- Автопубликация при пуше в `main`.
- Preview-деплои на PR — для ревью колонок перед публикацией.

### Производительность (для SEO)
- Lighthouse Score ≥ 95 по Performance.
- Нулевой JS по умолчанию (Astro).
- Картинки в WebP/AVIF, lazy-loading.
- Static generation всех страниц.

## 3. LLM для агентов

Архитектура построена на **трёх уровнях моделей** — это даёт экономию в 3–5× против «всё на Claude» без потери качества на критичных задачах. Полное обоснование и распределение по этапам — в `03_Архитектура_агентов.md`, раздел 3.12.

| Уровень | Модель | Где используется | Почему |
|---|---|---|---|
| **Tier 1: массовый/дешёвый** | **GLM-4-Flash** (free tier) | Pre-filter, embeddings, Enricher, QA | Бесплатно, много вызовов, простые задачи |
| **Tier 2: рабочая лошадка** | **DeepSeek-V3** | Translator, Author News, Author Digest, кластеризация в Filter+Dedup | Отличное качество для цены, в 12-13× дешевле Claude. Лучший переводчик с en/zh |
| **Tier 2+: для длинных текстов** | **DeepSeek-Reasoner (R1)** | Translator для текстов >8K токенов, сложные дайджесты | Лучший контекст для длинных материалов |
| **Tier 3: премиум** | **Claude Sonnet** | Author Review, эскалации с DeepSeek (когда QA FAIL) | Качество критично, материалов мало |
| **Tier 3: флагман** | **Claude Opus** | Author Reports (структура и research) | Максимальное качество для флагманских материалов |

### Эскалация DeepSeek → Claude (правила)
- QA FAIL на DeepSeek-материале → повтор на Claude Sonnet.
- Материал помечен `priority: true` → сразу Claude Sonnet/Opus.
- Тема: крупный M&A или запуск продукта крупного вендора → Claude Sonnet.

### API-ключи (все три провайдера)
- **Anthropic API key** — в `.env` в репо `media-agents` (GitHub Secrets).
- **DeepSeek API key** — аналогично. Создать на platform.deepseek.com.
- **GLM (Zhipu) API key** — аналогично. Создать на open.bigmodel.cn.

### Бюджет (оценка при 20–30 материалах в день)
- **Целевой бюджет:** $10–15/мес.
- **При «всё на Claude Sonnet»:** $40–60/мес.
- **Экономия:** 3–5×.
- **Таргет на 1 материал:** $0.30–0.50.

Мониторинг — через `data/state/llm_usage.jsonl`, куда пишется каждый вызов с моделью, токенами и стоимостью. Раз в неделю — сводный отчёт в AI sreda для корректировки распределения.

## 5. Оркестрация агентов

### Запуск
- **GitHub Actions** с расписанием (cron) — бесплатно для публичных репо, 2000 минут/мес для приватных.
- Альтернатива — любой VPS с cron (Hetzner ~$5/мес, Beget ~300 ₽/мес).
- Расписание:
  - Сборщик — каждые 2–4 часа.
  - Автор новостей — 2 раза в день (утром и вечером).
  - Автор дайджестов — раз в неделю.

### Логи и наблюдаемость
- Логи в GitHub Actions artifacts или в простой SQLite/Postgres.
- Алерты в Telegram-бот при ошибке агента.

## 7. Что вы делаете в Claude Code

На основе этого и других ТЗ, в Claude Code вы реализуете:
1. Инициализацию Astro-проекта с Tailwind и Content Collections.
2. Шаблоны страниц: главная, рубрика, статья, автор, статические.
3. Компоненты: карточка статьи, hero, навигация, футер, форма подписки.
4. Параллельно — репозиторий `media-agents` с пайплайном сбора → фильтрации → написания → публикации.

В AI sreda мы здесь ведём **проектирование, правила для агентов, контент-стратегию, ревью материалов**.
