## Итерация 1: Site scaffold (1-2 ч)

**Репо:** `media-site`
**ТЗ-референсы:** `02_Структура_и_дизайн_сайта.md`, `04_Тех_стек_и_интеграции.md`

### Команды для Claude Code

```
# Контекст для Claude Code:
# Прочитай 02_Структура_и_дизайн_сайта.md и 04_Тех_стек_и_интеграции.md
# Создай Astro-проект с Tailwind и Content Collections
```

### Задачи

1. `npm create astro@latest . -- --template minimal --install`
2. `npx astro add tailwind --yes`
3. `npm install @fontsource/source-serif-pro @fontsource/inter`
4. Настроить `tailwind.config.mjs`:
   * Цвета из 02, раздел 7 (`--color-bg`, `--color-accent`, категории)
   * Шрифты: serif для тела, sans для заголовков
   * Custom spacing (8px base)
5. Настроить `astro.config.mjs`:
   * `site: 'https://[domain].ru'`
   * Integrations: tailwind
   * Markdown: shiki theme
6. Создать `src/content/config.ts` — Content Collections:

```typescript
// Схема из 01_Контент-стратегия.md и 03_Архитектура_агентов.md
import { defineCollection, z } from 'astro:content';

const articles = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    description: z.string().max(160),
    pubDate: z.coerce.date(),
    modDate: z.coerce.date().optional(),
    type: z.enum(['news', 'digest', 'review', 'column', 'report']),
    category: z.enum(['adtech-ru', 'adtech-world', 'adtech-asia', 'market-news', 'tools', 'creative', 'reports']),
    geo: z.array(z.enum(['ru', 'world', 'asia'])).default([]),
    tags: z.array(z.string()).default([]),
    author: z.string().optional(),
    featured: z.boolean().default(false),
    cover: z.string().optional(),
    source: z.object({
      title: z.string(),
      url: z.string().url()
    }).optional(),
  })
});

const authors = defineCollection({
  type: 'content',
  schema: z.object({
    name: z.string(),
    role: z.string(),
    bio: z.string(),
    photo: z.string().optional(),
    social: z.object({
      linkedin: z.string().optional(),
      telegram: z.string().optional(),
      twitter: z.string().optional(),
    }).optional(),
  })
});

export const collections = { articles, authors };
```

1. Создать структуру каталогов:
   ```
   src/
   ├── content/
   │   ├── config.ts
   │   ├── news/
   │   ├── digest/
   │   ├── reviews/
   │   ├── columns/
   │   ├── reports/
   │   └── authors/
   ├── layouts/
   │   └── BaseLayout.astro
   ├── components/
   ├── pages/
   │   └── index.astro
   └── styles/
       └── global.css
   ```
2. Базовый `BaseLayout.astro`:
   * `<html>`, `<head>` с meta-тегами
   * `<slot />` для контента
   * Подключение шрифтов через Fontsource
   * Tailwind global styles
3. Простая `index.astro` — заглушка «Hello» для проверки.

### Критерий приёмки

* `npm run dev` запускается без ошибок
* Tailwind работает (проверить: `bg-accent` рендерится)
* Content Collections типизированы (создать тестовый `.md` — проходит валидацию)
* `npm run build` собирается без ошибок

## Итерация 5: Agents — автор + enricher + QA (2-3 ч)

**Репо:** `media-agents`
**ТЗ-референсы:** `03_Архитектура_агентов.md` (разделы 3.5, 3.9, 3.10, 3.11), `06_SEO_GEO_и_AI-разметка.md`

### Задачи

1. **`llm_client.py`** — абстракция над провайдерами:
   ```python
   class LLMClient:
       def __init__(self, provider: str): ...
       def chat(self, messages: list, **kwargs) -> str: ...

   # DeepSeekClient, GLMClient, ClaudeClient
   ```
2. **`news_writer.py`** (03, раздел 3.5):
   * Модель: DeepSeek-V3
   * Вход: переведённый `curated_item`
   * Промпт с принципом «анти-ИИ-голос»
   * Выход: Markdown с front-matter
   * Минимум 500 знаков с пробелами
   * Эскалация: QA FAIL → Claude Sonnet
3. **`enricher.py`** (03, раздел 3.9):
   * Модель: GLM-4-Flash (или DeepSeek, если GLM недоступен)
   * Генерация slug, description, тегов
   * Проверка по чек-листу (06, раздел 11)
4. **`qa.py`** (03, раздел 3.10):
   * rule-based: схема front-matter, длина, slug уникальность
   * Проверка: ≥ 500 знаков, title 50-80 симв, description ≤ 160 симв
   * Анти-ИИ-паттерны: regex на запрещённые клише
   * Выход: PASS / FAIL с причиной
5. **`git_publisher.py`** (03, раздел 3.11):
   * Коммит Markdown в `media-site/src/content/news/`
   * Push в `main` (для news)
   * Лог в `data/published.jsonl`
6. **Оркестратор** — `run_pipeline.py`:
   ```
   python run_pipeline.py --dry-run     # до конца, без git push
   python run_pipeline.py               # полный пайплайн: collect → translate → write → enrich → qa → publish
   python run_pipeline.py --test        # пушит в test-ветку
   ```

### Критерий приёмки

* `python run_pipeline.py --dry-run` производит 1-3 готовых Markdown-файла
* Файлы проходят QA (PASS)
* Контент ≥ 500 знаков, нет ИИ-клише
* `python run_pipeline.py` коммитит в `media-site` → Vercel деплоит
* На сайте появляются 1-3 авто-новости

## Итерация 6: Pipeline + cron (1-2 ч)

**Репо:** `media-agents`
**ТЗ-референсы:** `03_Архитектура_агентов.md` (разделы 5, 7), `04_Тех_стек_и_интеграции.md` (раздел 5)

### Задачи

1. **GitHub Actions workflow** (`.github/workflows/agents.yml`):
   ```yaml
   name: Media Agents Pipeline
   on:
     schedule:
       - cron: '0 7,13 * * *'  # 10:00 и 16:00 МСК
     workflow_dispatch: {}

   jobs:
     pipeline:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-python@v5
         - run: pip install -r requirements.txt
         - run: python run_pipeline.py
           env:
             DEEPSEEK_API_KEY: ${{ secrets.DEEPSEEK_API_KEY }}
             ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
             GLM_API_KEY: ${{ secrets.GLM_API_KEY }}
   ```
2. **GitHub Actions: checkout media-site** — для git push нужен доступ к второму репо (deploy key или token).
3. **Алерты** — `utils/telegram_notifier.py` (из `adtechreview/scripts/telegram_bot.py`):
   * Успех: «Опубликовано N новостей»
   * Ошибка: «Pipeline FAIL: \[причина]»
4. **Мониторинг** — `scripts/cost_report.py`:
   * Читает `data/state/llm_usage.jsonl`
   * Выводит недельный отчёт по моделям и стоимости

### Критерий приёмки

* GitHub Actions триггерится по расписанию
* Pipeline отрабатывает end-to-end без ручного вмешательства
* На сайте появляются свежие новости каждый день
* Алерты приходят в Telegram
* `cost_report.py` показывает ≤ $0.50 на материал

## Зависимости между итерациями

```
Итерация 0 (Setup)
    ↓
Итерация 1 (Site scaffold) ←── Итерация 4 (Agents: сбор)
    ↓                              ↓
Итерация 2 (Pages) ←────────── Итерация 5 (Agents: автор)
    ↓                              ↓
Итерация 3 (Design + deploy) ←── Итерация 6 (Pipeline)
    ↓                              ↓
         Итерация 7 (Контент)
```

**Параллельность:** Итерации 1-3 (сайт) и 4-5 (агенты) можно делать параллельно в разных Claude Code-сессиях. Итерация 6 требует, чтобы оба репо были готовы.

## Что делает AI sreda между итерациями

* **Ревью ТЗ** после каждой итерации — корректировка промптов, правил, метрик
* **Ресёрч** новых источников, конкурентов, инструментов
* **Анализ качества** первых авто-новостей → тюнинг промптов
* **Контент-планирование** — темы для обзоров, колонок, отчётов
* **Документирование решений** — фиксация архитектурных выборов
