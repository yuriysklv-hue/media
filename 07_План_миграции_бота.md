# План миграции: adtechreview bot -> media-agents

> Картирование существующего кода из `/Kojori/adtechreview/` на новую архитектуру `media-agents`. Цель — переиспользовать валидированные компоненты и не писать с нуля то, что уже работает.

***

├── run_daily_digest.py      ← CLI runner (--test, --dry-run, --no-confirm)

## 4. Что переписывается с нуля

Этого в боте не было:

1. **Pre-Filter на GLM-4-Flash** — бот использовал keyword-скоринг, новый добавляет LLM-классификацию поверх.
2. **Filter + Dedup с embeddings** — бот дедуплицировал только по article_id. Новая — семантическая дедупликация через GLM embeddings + cosine similarity.
3. **Author News** — бот не писал отдельные новости, только дайджесты. Новая — самостоятельные short-form материалы.
4. **Author Review / Author Reports** — бот не делал обзоров и исследований. Полностью новое.
5. **Enricher (SEO/Geo)** — боту не нужен был SEO. Новая — генерация slug, description, тегов.
6. **QA / Validator** — бот не валидировал формат. Новая — проверка схемы, длины, фактов, анти-ИИ-паттернов.
7. **Git Publisher** — бот публиковал в Telegram. Новая — коммиты в git-репозиторий сайта.
8. **LLM-абстракция** — бот работал только с DeepSeek. Новая — мультипровайдерный клиент (DeepSeek + GLM + Claude).

## 5. Порядок миграции (по этапам)

### Этап 0: Форк и рефакторинг (1 день)

```
1. Создать репозиторий media-agents
2. Скопировать из adtechreview:
   - hybrid_generator.py → временное media-agents/_legacy/
   - config.yaml → media-agents/config.yaml (затем расширить)
   - ideal_digest_format.md → media-agents/docs/
3. Установить зависимости: feedparser, requests, beautifulsoup4, pyyaml
4. Добавить новые: anthropic, openai (для DeepSeek), zhipuai (или requests)
5. Проверить, что старый код запускается в новом репо
```

### Этап 1: RSS + Scoring (1 день)

```
1. Вынести RSS-парсинг в collectors/rss_collector.py
2. Вынести скоринг в utils/scoring.py (прямой перенос словарей)
3. Вывод — в inbox/raw_items.jsonl (новый формат)
4. Тест: собрать сырьё с 5 источников, проверить скоринг на 50 items
5. Сравнить score с результатами старого бота (debug_scoring.py)
```

### Этап 2: Translator (0.5 дня)

```
1. Вынести переводчик в processors/translator.py
2. Перенести промпт из hybrid_generator.py
3. Расширить правила (см. 03, раздел 3.3)
4. Добавить кэш: state/translations_cache.json
5. Тест: перевести 20 raw_items, сверить качество со старым ботом
```

### Этап 3: Author News (1 день)

```
1. Создать writers/news_writer.py
2. Промпт из 03, раздел 3.5
3. Вход: curated_item (уже переведённый), выход: Markdown с front-matter
4. Тест: написать 10 новостей, проверить front-matter и анти-ИИ-голос
```

### Этап 4: Publisher + QA (1 день)

```
1. Создать publishers/git_publisher.py
2. QA: валидация front-matter через jsonschema
3. Тест: закоммитить 5 новостей в test-ветку media-site
4. Проверить, что Vercel задеплоил
```

### Этап 5: Enricher + Orchestrator (1 день)

```
1. Создать processors/enricher.py (GLM-Flash)
2. Создать orchestrator.py (замена run_daily_digest.py)
3. Флаги: --test, --dry-run, --source rss, --limit N
4. GitHub Actions workflow с cron
5. End-to-end тест: RSS → score → translate → write → enrich → QA → publish
```

## 6. Что НЕ мигрирует (намеренно)

| Компонент | Причина |
|---|---|
| **Outreach/** (лидогенерация) | Отдельный модуль, не нужен для MVP-1. Рассматривать в Релизе 4. |
| **Азия/Pacific RSS-источники** | Проверено — низкое качество. Покрытие Азии через vendor press releases. |
| **Публикация контента в Telegram** | Новый канал публикации — сайт. Telegram остаётся только для алертов. |
| **Stateless-режим** | Бот не хранил состояние между запусками. Новый — персистентные JSONL + seen_urls. |
| **Single-LLM архитектура** | Бот работал только с DeepSeek. Новый — 3 провайдера с эскалацией. |

## 7. Риски миграции

| Риск | Вероятность | Митигация |
|---|---|---|
| Качество новостей на DeepSeek ниже, чем дайджестов | Средняя | A/B тест на 20 материалах DeepSeek vs Claude. Эскалация при FAIL. |
| Скоринг на новых источниках (RU) ведёт себя иначе | Высокая | Отдельный скоринг для RU-источников, тюнинг после первых 30 публикаций. |
| Git publisher конфликтует при частых коммитах | Низкая | Один пуш за цикл (batch), а не на каждую новость. |
| GLM-4-Flash лимиты на free tier | Средняя | Fallback на keyword-скоринг (из бота) при недоступности GLM. |
| Перевод RU→RU (Adindex, Sostav) пропускается корректно | Низкая | Проверка `language == "ru"` в Translator — пропуск. |
