## Этап 2. MVP-1 агентов (09–13.07, \~5 дней)

### Задачи (через Claude Code в репо `media-agents`)

* [ ] Инициализировать Python-проект, виртуальное окружение, зависимости.
* [ ] Создать структуру:
  ```
  media-agents/
  ├── src/
  │   ├── collectors/      # сборщики по источникам
  │   │   ├── rss.py
  │   │   ├── press_releases.py
  │   │   ├── vendor_blogs.py
  │   │   └── earnings.py
  │   ├── processors/      # фильтрация, дедупликация
  │   ├── writers/         # LLM-написание
  │   │   ├── news_writer.py
  │   │   ├── digest_writer.py
  │   │   └── review_writer.py
  │   ├── publishers/      # публикация в git
  │   │   └── git_publisher.py
  │   ├── prompts/         # шаблоны промптов для каждого агента
  │   ├── models/          # дата-классы для Article, Source, RawItem
  │   └── config.py
  ├── data/
  │   ├── inbox/           # сырые данные от сборщиков
  │   ├── drafts/          # черновики от LLM
  │   └── published/       # опубликованные (для дедупликации)
  ├── .github/workflows/   # GitHub Actions cron
  ├── .env.example
  └── README.md
  ```

* [ ] Реализовать **Фильтр+дедупликатор**: хеш заголовка + embeddings для семантической дедупликации.
* [ ] Реализовать **Агента-автора новостей** (Claude Sonnet) — берёт пул сырых новостей, пишет 3–5 коротких новостей на русском.
* [ ] Реализовать **Агента-публикатора**: git push в `media-site/src/content/news/`.
* [ ] Настроить **GitHub Actions** с cron (2 раза в день).
* [ ] Подключить старые наработки по парсингу (переиспользовать XPath/селекторы).

## Риски и митигация

| Риск                               | Вероятность | Митигация                                                                      |
| ---------------------------------- | ----------- | ------------------------------------------------------------------------------ |
| Не успеваете за 1–2 недели         | Высокая     | Срезать скоуп: меньше рубрик, меньше шаблонов. Главное — 10 материалов.        |
| Качество агентских новостей низкое | Высокая     | На старте жёсткий гибридный контроль. Гонять промпты, пока не stabilizes.      |
| Источники банят парсинг            | Средняя     | RSS и официальные блоги — обычно ок. Scraping — с задержками и UA-заголовками. |
| Домен с плохой историей            | Низкая      | Перед покупкой проверять через Wayback Machine, Ahrefs DR, Spam Score.         |
| Перерасход на API LLM              | Средняя     | Дешёвые модели для фильтрации, лимиты на количество вызовов в день.            |
| Выгорание (один человек)           | Высокая     | Жёстко приоритизировать. Лучше меньше, но стабильно.                           |
