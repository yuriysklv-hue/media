## Что это

Conversions API (CAPI) — это серверный интерфейс (API), разработанный компанией Meta\* (ранее Facebook\*), который позволяет рекламодателям передавать данные о действиях пользователей (конверсиях) напрямую с собственного сервера или бэкенда на серверы рекламной платформы. В отличие от традиционного пиксельного (браузерного) трекинга, CAPI не зависит от работы файлов cookie, блокировщиков рекламы и ограничений браузеров (например, Intelligent Tracking Prevention от Apple). Данные передаются по защищённому каналу, что повышает надёжность и точность учёта конверсий, особенно в условиях ужесточения политик конфиденциальности.

## Как работает

Принцип работы Conversions API основан на прямой серверной интеграции. Когда пользователь совершает целевое действие на сайте (покупку, регистрацию, подписку) или в приложении, сервер рекламодателя формирует HTTP-запрос с данными о событии (тип конверсии, время, сумма заказа, идентификатор пользователя и т.д.) и отправляет его на эндпоинт API Meta\*. Для сопоставления события с конкретным пользователем и рекламным объявлением используются идентификаторы: хешированные email, телефон, Facebook\*-пиксельный ID (если пиксель также установлен) или браузерный cookie (через параметр `event_source_url`). CAPI может работать как самостоятельно, так и в паре с пикселем (режим «deduplication»), когда данные дублируются, а платформа автоматически удаляет дубликаты на основе общего идентификатора. Это позволяет повысить точность атрибуции и улучшить модели машинного обучения для оптимизации рекламных кампаний.

## Для кого

Conversions API предназначен для рекламодателей, которые:
- Используют рекламные платформы Meta\* (Facebook\*, Instagram\*) и сталкиваются с потерей данных о конверсиях из-за блокировщиков рекламы или браузерных ограничений.
- Работают с большими объёмами транзакций и нуждаются в максимально точной атрибуции (например, e-commerce, финтех, гемблинг).
- Имеют собственную серверную инфраструктуру и техническую возможность для интеграции (разработчики, DevOps).
- Хотят передавать данные о конверсиях, которые происходят не на сайте (например, в мобильном приложении, по телефону, в офлайн-точках), через серверный канал.
- Стремятся соответствовать требованиям регуляторов (например, 152-ФЗ «О персональных данных» в РФ) и минимизировать передачу данных через браузер.

## Альтернативы

В качестве альтернатив Conversions API от Meta\* можно рассматривать:
- **Google Ads Conversion Tracking (серверный)** — аналогичный серверный API от Google для передачи данных о конверсиях в Google Ads. Позволяет отправлять события с сервера, минуя браузерные ограничения.
- **Yandex.Metrica API (серверная передача целей)** — через API Яндекс.Метрики можно передавать данные о целях (конверсиях) напрямую с сервера, что актуально для рекламодателей, использующих Яндекс.Директ.
- **Server-side GTM (Google Tag Manager)** — позволяет развернуть серверный контейнер GTM, который принимает данные с сайта и перенаправляет их в различные рекламные системы (включая Meta\* и Google) через серверные теги, обеспечивая единую точку интеграции и контроль над данными.

\* Meta, Facebook и Instagram принадлежат Meta Platforms, которая признана в России экстремистской организацией; её деятельность в РФ запрещена.
