Яндекс ввёл в бета-эксплуатацию открытый API для своей ecommerce-платформы KIT — инструмент, который позволяет подключать к магазину любые внешние системы учёта, от ERP до PIM, и автоматизировать рутину вроде обновления остатков или цен. Раньше такие интеграции были доступны только через штатные модули платформы, теперь — через единый программный интерфейс.

API пригодится прежде всего технологическим партнёрам, агентствам и компаниям, которые ведут собственный товарный учёт и хотят синхронизировать его с магазином без ручного переноса данных. Через API можно не только загружать товары и обновлять остатки после каждой продажи, но и передавать заказы в CRM, корректировать цены при изменении в учётной системе и настраивать вебхуки для уведомлений о событиях. Для магазинов с большим ассортиментом и несколькими складами это прямой путь к сокращению числа ошибок и высвобождению времени сотрудников.

На старте API охватывает 17 разделов магазина: от каталога и складов до заказов, клиентов, скидок и промокодов. В планах — расширять список, причём Яндекс принимает пожелания от пользователей через поддержку. Для подключения достаточно сгенерировать токен в личном кабинете — правда, сохранить его нужно самостоятельно: повторно посмотреть значение токена нельзя.

Технические детали: API работает по UTC, на один магазин — не более 10 запросов в секунду. Ограничение стандартное для платформенных решений, но тем, кто привык к массовым выгрузкам в пиковые часы, стоит закладывать тайм-ауты и ретраи. В остальном — никаких ограничений по стеку технологий: интерфейс REST, язык запроса не важен.

Пока инструмент в статусе бета, и главный вопрос — как быстро Яндекс будет дорабатывать API под реальные бизнес-сценарии. Судя по динамике обновлений и открытому каналу связи с разработчиками, ставка делается на гибкость. Для селлеров, которые уже сидят на KIT или только присматриваются, появление API — это снятие одного из главных барьеров при переходе с самописных или legacy-решений.
