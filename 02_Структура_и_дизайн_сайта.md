> ТЗ для разработки в Claude Code. На основе этого документа строится Astro-проект.
>
> ⚠️ **Актуальное направление дизайна (05.07.2026) — «тех/продукт минимал».** Часть решений ниже устарела; приоритетны канонический макет `mockups/1screen-mockup-minimal.html` и раздел «Зафиксированные дизайн-решения» в `CLAUDE.md`. ## 1. Референсы и позиционирование дизайна

### На что опираемся

* **Digiday** (digiday.com) — типографика, плотность ленты, формат карточек новостей, разделение по типам контента.
* **The Drum** (thedrum.com) — общий стиль подачи, микс новостей и аналитики, оформление креативных кейсов.
* **Detailed.com** (detailed.com) — раздел Reports (глубокие аналитические лонгриды с характерной версткой).
* **Adindex** (adindex.ru) — нормальная типографика для русскоязычного B2B-медиа.
* **Hirehi.ru** (не про рекламу) — раздел сервисов и инструментов, авторизация (UX-референс для будущих b2b-сервисов).

### Чего избегаем

* Стиля VC.ru (визуально и структурно).
* Перегруженности баннерами.
* «Пресс-релизного» стиля cossa.

### Общее ощущение

Журнальное, экспертное, чистое, с акцентом на типографику. Не «блог», не «лента новостей», а **профессиональное медиа**.

## 3. Главная страница

### Структура (сверху вниз)

1. **Header** (sticky, тонкий) — лого, навигация, иконка поиска, CTA «Подписаться».
2. **Hero** — большой featured-материал (по флагу `featured: true`):
   * Обложка (16:9 или 4:3).
   * Категория-плашка.
   * Заголовок (H1, крупный).
   * Лид (1–2 строки).
   * Автор + дата + время чтения.
3. **Top-3 вторичных материала** — сетка 3 колонки, без обложек или с маленькими.
4. **Лента «Свежее»** — вертикальный список карточек:
   * Каждая карточка: маленькая обложка слева, контент справа (категория, заголовок, лид, мета).
   * Пагинация или infinite scroll (infinite — в MVP-2).
5. **Блок «Дайджест недели»** — выделенный блок с последним дайджестом.
6. **Блок «Мнения»** — 2–3 последние колонки с акцентом на авторов.
7. **Подписка на рассылку** — большой блок с email-формой и обещанием пользы.
8. **Footer** — рубрики, о нас, реклама, соцсети, копирайт.

### Принцип верстки

* На десктопе основная колонка 720–800px (для читаемости), сайдбар справа для топ-5 популярных / рекламы — опционально в MVP-2.
* На мобайле — одна колонка, hero уменьшается, карточки стекаются.

***

## 4. Страница категории (`/category/[slug]`)

### Структура

1. Header (общий).
2. H1 — название категории.
3. Описание категории (1–2 предложения, статичный текст).
4. Фильтры: по типу контента (новости / обзоры / колонки), по гео (РФ / Мир / Азия).
5. Лента материалов этой категории (тот же компонент ArticleCard).
6. Пагинация (по 20 на странице).

***

## 5. Страница материала (`/article/[slug]`)

Самая важная страница — на ней проводит время читатель.

### Структура

1. **Header** (общий, sticky).
2. **Шапка статьи:**
   * Категория-плашка (например, «Adtech Россия»).
   * Заголовок H1 (большой, до 56px на десктопе).
   * Лид под заголовком (italic или serif, серый).
   * Мета-строка: автор (ссылка на профиль) • дата • время чтения • теги.
3. **Обложка** — на всю ширину контентной колонки.
4. **Тело статьи** — типографика:
   * Ширина строки: 680–720px (для читаемости).
   * Шрифт: serif (Source Serif Pro / Lora / Literata), 18–20px, line-height 1.6.
   * Абзацы: отступ 1em.
   * H2/H3 — sans-serif, жирные, контрастные.
   * Цитаты — отдельным стилем с вертикальной чертой.
   * Списки — аккуратные, без избыточных отступов.
   * Ссылки — подчёркнутые, цвет акцент.
   * Картинки — с alt-текстом, подписью, в рамке или без.
   * Код (если есть) — моноширинный, в блоке.
5. **Источник** — отдельный блок в конце: «Источник: [title](url)».
6. **Теги** — кликабельные.
7. **Об авторе** — мини-карточка: фото, имя, должность, ссылка на профиль.
8. **Читайте также** — 3 связанных материала по тегам/категории.
9. **Подписка** — CTA в конце статьи.
10. **Footer** (общий).

### Спец-блоки в теле статьи (для редакторского контента)

* Callout (выделенная цитата).
* Stat block (крупная цифра + расшифровка).
* Comparison table (для обзоров инструментов).
* Image gallery (для креативных кейсов).

Эти блоки реализуются как Astro-компоненты или MDX-компоненты.

## 6. Страница автора (`/author/[slug]`)

### Структура

1. Header.
2. Hero автора:
   * Фото (квадрат, 200x200).
   * Имя (H1).
   * Должность / специализация.
   * Соцсети (LinkedIn, Twitter, Telegram).
   * Краткое био (3–5 предложений).
3. Лента материалов автора.
4. Контакты для связи (если публичные).

### Формат данных автора

```yaml
---
name: "Юрий Соколов"
slug: "y-sokolov"
role: "Основатель медиа, экс-руководитель по монетизации 2ГИС"
bio: "20+ лет в рекламе и adtech. Запускал рекламные продукты..."
photo: "/images/authors/y-sokolov.jpg"
social:
  linkedin: "..."
  telegram: "@..."
  twitter: "..."
---
```

## 7. Дизайн-система

### Цветовая палитра

**Принцип:** нейтральная основа + один акцентный цвет. Не пёстро.

```
/* Базовые */
--color-bg: #FFFFFF           /* основной фон */
--color-bg-alt: #F7F7F5       /* фон блоков, alternate sections */
--color-text: #1A1A1A         /* основной текст */
--color-text-soft: #4A4A4A    /* вторичный текст */
--color-text-muted: #888888   /* мета-инфо, даты */
--color-border: #E5E5E0       /* границы, разделители */

/* Акцент (подобрать) */
--color-accent: #C8472B       /* например, тёплый red-orange (как Digiday) */
--color-accent-hover: #A53A22

/* Для плашек категорий */
--color-cat-ru: #2E5C8A       /* синий для RU */
--color-cat-world: #2D8A5F    /* зелёный для Мира */
--color-cat-asia: #8A2D5C     /* бордовый для Азии */
--color-cat-tools: #8A6F2D    /* охра для Инструментов */

/* Темная тема (MVP-2) */
--color-bg-dark: #0F0F0F
--color-text-dark: #E5E5E0
```

**Решить:** акцентный цвет подобрать вместе с дизайнером или зафиксировать сейчас. Я предлагаю начать с **тёплого red-orange** (как Digiday / The Drum), можно изменить позже.

### Типографика

| Элемент               | Шрифт                           | Размер (десктоп) | Размер (мобайл) |
| --------------------- | ------------------------------- | ---------------- | --------------- |
| Заголовок статьи (H1) | Serif (Source Serif Pro / Lora) | 48–56px / 1.15   | 28–32px         |
| H2 в статье           | Sans (Inter / Manrope)          | 28–32px / 1.25   | 22–24px         |
| H3                    | Sans                            | 22–24px          | 18–20px         |
| Тело статьи           | Serif                           | 18–20px / 1.6    | 16–17px / 1.6   |
| Лид                   | Serif italic                    | 22–24px          | 18–20px         |
| Мета (дата, автор)    | Sans                            | 13–14px          | 12–13px         |
| Заголовок карточки    | Sans semibold                   | 18–22px          | 16–18px         |
| Лид на карточке       | Sans                            | 14–15px          | 13–14px         |

**Шрифты подключить через Fontsource (для self-hosting, без Google Fonts):**

* `@fontsource/source-serif-pro` (400, 600, 700)
* `@fontsource/inter` (400, 500, 600, 700)
* Кириллица проверяется для каждого шрифта (Source Serif Pro и Inter поддерживают).

### Сетка и отступы

* Max-width контейнера: 1200px.
* Контентная колонка статьи: 720px.
* Базовый отступ: 8px (использовать Tailwind spacing scale: 1 \= 8px).
* Ритм: 8/16/24/32/48/64/96.

### Радиусы

* Карточки: 8px (subtle).
* Кнопки: 4px (или pill для CTA).
* Изображения: 4–8px.

### Тени

* Минимальные. Только для sticky-элементов и hover-состояний.

## 9. Адаптивность

Брейкпоинты (Tailwind default):

* `sm` 640px — большие мобайлы.
* `md` 768px — планшеты.
* `lg` 1024px — маленький десктоп.
* `xl` 1280px — стандартный десктоп.

Принцип: **mobile-first**. На мобайле — одна колонка, упрощённая навигация, hero без крупных обложек.

***

## 10. SEO-требования

### На каждой странице

* `<title>` — заголовок материала + « | Название медиа».
* `<meta name="description">` — из front-matter `description`.
* Open Graph: `og:title`, `og:description`, `og:image`, `og:url`, `og:type=article`.
* Twitter Card: `summary_large_image`.
* Canonical URL.

### На статье

* schema.org **NewsArticle** или **Article**:
  * headline, datePublished, dateModified, author, image, publisher, articleBody.
* Breadcrumbs (schema.org BreadcrumbList).

### Глобально

* `sitemap.xml` — генерируется через `@astrojs/sitemap`.
* `robots.txt` — разрешаем всё, ссылка на sitemap.
* Структура URL: человекопонятные slug-и (translit или английский slug).

## 11. Производительность

* Lighthouse Performance ≥ 90 (target 95+).
* Нулевой JS по умолчанию (Astro).
* Шрифты: `font-display: swap`, preload критических.
* Картинки: WebP/AVIF, `loading="lazy"` ниже первого экрана.
* Inline критического CSS.

***

## 12. Доступность (a11y)

* Контрастность текста — AA как минимум (4.5:1).
* Все интерактивные элементы доступны с клавиатуры.
* Alt-тексты у картинок.
* ARIA-роли у динамических элементов (поиск, меню).
